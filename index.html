Correcto uso de ramas ->

Para usar las ramas correctamente utilizamos la metodologia git Flow.

Primer paso:

Creamos el repositorio en github donde el usuario 1 realizara la inicialazion del proyecto.

Tendremos la rama master como primera rama.

Segundo paso:

AÃ±adiremos una segunda rama que es con la que desarrollaremos cada punto de la aplicacion.

Cuando un usuario tenga que implementar algo creara una rama feature/{nombre de lo implementado} y trabajara en esa rama.

Al terminal la implementacion creara un merge con develop y lo subira al repositorio remoto.

Tercer paso:

Cuando tengamos una version estable en la rama develop realizaremos un merge de develop con master y tendremos la version en produccion.
